# Roadmap

Public roadmap for JQ AI Skills.

This roadmap is intentionally public-safe. It does not include client names, private project plans, internal logs, raw prompts, account details, or unpublished implementation notes.

The stable 2026 social preview is not updated for every release.

## Now

The current focus is adoption: make the library easier to inspect, install, request, and improve.

- Keep the first-use path clear from `START_HERE.md` to install, troubleshooting, support, and first run.
- Maintain the skill quality matrix so visitors can compare maturity, examples, safety sensitivity, and install notes.
- Keep issue templates focused on public-safe reports and requests.
- Keep release notes, changelog entries, and website copy aligned with the latest public release.

## Next

Near-term work should make the public library easier to trust and reuse.

- Add more fake-data examples for the most useful skill bundles.
- Expand install and verification notes for common Codex and Claude Code setups.
- Improve the catalog with clearer use-case groupings.
- Add more public-safe before and after samples for content, research, and release workflows.
- Keep the website, profile README, release pages, and repository docs aligned.

## Later

Longer-term work can grow the library once the adoption path stays steady.

- Add new public-safe skills when they solve a repeatable workflow.
- Add more demo walkthroughs for skill bundles, not just individual skills.
- Publish contribution examples that show how to propose a skill without exposing private material.
- Improve automated checks around skill metadata and public-safety review.

## Not Planned

These are outside the public direction of this repo.

- Unattended sending, scraping, or publishing without human review.
- Workflows that require posting secrets, tokens, client data, private exports, or account details.
- Raw private prompts, chat logs, memory files, production data, or internal operating procedures.
- Claims of platform compatibility that are not backed by install or usage evidence in the repo.

## Request Something

Use the skill request issue template when you want to propose a new public-safe skill. Keep examples fictional or generalized, and do not include credentials, client material, private links, or local machine paths.
