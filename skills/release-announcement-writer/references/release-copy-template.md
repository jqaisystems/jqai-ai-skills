# Release Copy Template

Use this template when the user wants a complete launch pack.

## GitHub Release Notes

```markdown
# <Version>: <Plain-English Release Name>

<One short paragraph: what changed and why it matters.>

## Highlights

- <User-facing change and benefit.>
- <User-facing change and benefit.>
- <User-facing change and benefit.>

## Install or Update

<Command, folder copy instruction, package install note, or link to docs.>

## Links

- Repo: <url>
- Release: <url>
- Demo or case study: <url>
```

## Website / README Blurb

```text
<Project/system name> now includes <specific change>. It helps <audience> <outcome> by <mechanism>. See <link>.
```

## Social Post

```text
New release: <project/system name> <version>.

<What changed, in one sentence.>

<Why it matters, in one sentence.>

Try it here:
<url>
```

## Changelog Groups

Use these headings only when they have real content:

- Added
- Changed
- Fixed
- Docs
- Safety
- Removed

## Verification Checklist

- Version and date match the release tag.
- Links open to public pages.
- Install steps match the current README.
- Screenshots/GIFs are public-safe.
- No private paths, secrets, client names, internal logs, or raw prompts are included.
