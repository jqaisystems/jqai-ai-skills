# Safety Boundary Examples

Use one of these patterns and adapt it to the system.

## Generic Automation

This public case study excludes source code, private prompts, credentials, databases, logs, exports, deployment details, and any private client or account data.

## Outreach Or Lead Workflow

This public case study excludes source code, private prompts, prospect lists, enrichment exports, email addresses, phone numbers, API credentials, scoring prompt details, databases, logs, and live outreach configuration.

## Content Or Publishing Pipeline

This public case study excludes source code, private prompts, unpublished drafts, account credentials, scheduling tokens, platform API details, logs, analytics exports, and any client-owned content that has not been approved for public use.

## Client Intake Or CRM Workflow

This public case study excludes source code, private prompts, form submissions, lead records, CRM data, email addresses, phone numbers, email credentials, delivery logs, and private client or prospect communication.

## Research Or Briefing Workflow

This public case study excludes source code, private prompts, paid research exports, private notes, account credentials, source databases, raw transcripts, logs, and any confidential business strategy.

## AI Agent Or Operations System

This public case study excludes source code, memory files, session logs, local command paths, private prompts, credentials, deployment details, internal dashboards, databases, and any system-of-record data.

## Approval Language

Use one of these sentences when relevant:

- Nothing is sent, published, scheduled, or changed without human approval.
- The system drafts and prioritizes work, but the human decides what moves forward.
- AI output enters a review queue before it reaches a client, customer, or public channel.
- The workflow is designed for decision support, not unattended execution.
