# v0.6.7

JQ AI Skills now has a first skill walkthrough for visitors who have chosen one skill and want to run a safe fake-data test before touching real work.

This release keeps the stable 2026 social preview unchanged. The update focuses on the step after scoring a first skill: showing a complete first run with `github-safe-publisher`, fake files, expected verdicts, and review gates.

## What Changed

- Added `docs/examples/first-skill-walkthrough.md`.
- Added a complete fake-data run for choosing, reviewing, temporarily installing, prompting, and interpreting `github-safe-publisher`.
- Linked the walkthrough from the README, `START_HERE.md`, `QUICK_REFERENCE.md`, public proof index, visitor paths, examples index, evaluation checklist, first skill scorecard, skill selection guide, catalog, quick reference walkthrough, first-run sample, and changelog.
- Updated the README release badge to `v0.6.7`.

## Why It Matters

The repo already helps visitors evaluate the library and choose one safe first install.

The walkthrough closes the next practical gap: it shows what to do immediately after choosing the skill, using fake material and a review-first publishing boundary.

## Safety Note

Use fake or public-safe input for the first run. Do not test with credentials, account data, logs, exports, customer records, screenshots, unpublished prompts, private project notes, or local private paths.
