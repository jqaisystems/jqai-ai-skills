# v0.4.5

JQ AI Skills now has a root `SECURITY.md` file.

This release adds a responsible-use and sensitive reporting policy for a public library of installable AI skill folders. It explains what belongs in a security report, how to avoid posting sensitive material in public issues, how to install skills safely, and how to keep private work out of public releases.

The stable 2026 social preview is unchanged.

## What's changed

- Added `SECURITY.md` as the root security and responsible-use policy.
- Linked `SECURITY.md` from the README and `START_HERE.md`.
- Updated `CHANGELOG.md` with the v0.4.5 entry.
- Updated the README release badge to `v0.4.5`.
- Kept the stable 2026 social preview unchanged.

## Why it matters

Reusable AI workflows need trust boundaries.

This policy makes those boundaries explicit: inspect skills before installing them, keep sensitive details out of public reports, review outputs before publishing, and use the release checklist and safety scanner for public artifacts.
