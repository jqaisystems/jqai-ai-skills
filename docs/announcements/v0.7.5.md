# v0.7.5

JQ AI Skills now includes a release-to-showcase handoff proof sample.

This release keeps the stable 2026 social preview unchanged. The update focuses on the repeatable public routine that happens after a GitHub release is published: profile alignment, website source update, host upload package, live verification, and project note cleanup.

## What Changed

- Added `docs/examples/release-showcase-handoff-sample.md`.
- Used a fictional Example Skill Library release to show a public-safe handoff from GitHub release to profile update, website update, host upload package, live verification, and final status note.
- Documented expected artifacts, upload package shape, live URL checks, cache-busted verification, and `READY`/`REVIEW`/`BLOCK` outcomes.
- Linked the sample from the README, `START_HERE.md`, `QUICK_REFERENCE.md`, examples index, visitor paths, public proof index, catalog, quality matrix, and changelog.
- Updated the quality matrix to `v0.7.5` and added the release handoff sample to the visible coverage signals.
- Updated the README release badge to `v0.7.5`.

## Why It Matters

The repo already proves install, update, utility output, and demo walkthrough paths. The next trust question is whether releases are carried through the full public showcase system without stale pages or unsafe upload packages.

The new sample makes that operating routine inspectable without exposing private hosting details. Visitors can see how a release should move from GitHub to profile copy, website files, host upload package, live verification, and final `READY` status.

## Safety Note

The handoff sample uses fictional release names, fictional URLs, and generic file paths. It does not include hosting credentials, account menus, admin URLs, analytics, deployment logs, local machine paths, raw prompts, private chat logs, client material, or unpublished operational notes.
