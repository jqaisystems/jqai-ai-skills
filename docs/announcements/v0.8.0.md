# v0.8.0

JQ AI Skills grows from 12 to 21 skills. This is the largest release so far: nine new skills covering client work, outreach, writing quality, notes, AI search visibility, and folder maintenance. All nine follow the same conventions as the rest of the library: self-contained folders, ask-once configuration files instead of hardcoded personal data, and report-before-acting defaults wherever a skill touches files.

Two new areas join the catalog: Notes & Productivity and AI Search.

## What's changed

### Added: client work and forms

- `php-form-mailer`: wires any HTML form to a two-email PHP handler (owner notification plus branded confirmation to the submitter) with no form SaaS. Includes a honeypot spam check, a mailto fallback, and a live curl test step. Configuration comes from a `form-mailer.config.md` the skill sets up on first run.
- `client-intake-builder`: generates a branded, single-file HTML client intake questionnaire. Four question banks (brand identity, web design, automation, pre-call brief), styled from the project's own CSS when available. Pairs with `php-form-mailer` for the email backend.

### Added: outreach and sales

- `cold-outreach-writer`: drafts one personalised cold email under 180 words from a research block, plus three alternative subject lines, a LinkedIn note, and a 7-day follow-up. Drafts only, never sends, never fabricates claims. Sender profile lives in `outreach-profile.md`.
- `lead-scorer`: ranks a lead list 0 to 100 into hot, warm, and cold tiers against a user-defined rubric (`lead-scoring-rubric.md`), with a reason and next action per lead. Leads with thin data get flagged for research instead of guessed scores.

### Added: writing quality

- `brand-voice-linter`: lints copy against a `voice-guide.md`: buzzword clusters, hedge language, vague superlatives, passive voice, exclamation marks, and AI-tell phrases. Reports every violation with a suggested rewrite; fixes only on request. The bigger sibling of `em-dash-remover`.

### Added: notes and productivity

- `obsidian-daily-note`: creates or appends to today's daily note in an Obsidian vault from the terminal. Auto-detects the vault via `.obsidian`, respects the user's date format and folder, and is append-only by rule.
- `idea-inbox`: captures a raw idea as a dated, tagged markdown file and keeps an `IDEAS_INDEX.md` up to date. Cleans filler without compressing substance, and refuses to save ideas to the project root.

### Added: AI search and maintenance

- `llms-txt-generator`: generates a spec-compliant `llms.txt` so AI assistants and answer engines can understand and cite a site accurately. One honest line per page, private endpoints excluded, links validated, and diffs against an existing file instead of replacing it.
- `vault-janitor`: scans a folder tree for accumulated junk (dependency folders, build caches, stray backups, logs, OS litter) and produces a sized, grouped cleanup report. Report-only by default; archive-first even when a cleanup list is approved.
