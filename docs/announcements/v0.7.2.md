# v0.7.2

JQ AI Skills now includes an update-after-release proof trail.

This release keeps the stable 2026 social preview unchanged. The update focuses on what happens after the first install: pulling a new release, inspecting the changes, testing the refreshed skill copy in a temporary target, reinstalling the selected skill, reloading the tool, and running a fake check before using the refreshed skill on real work.

## What Changed

- Added `docs/examples/update-after-release-proof.md`.
- Linked the new proof trail from the README, `START_HERE.md`, `QUICK_REFERENCE.md`, public proof index, visitor paths, examples index, catalog, update guide, evaluation checklist, quality matrix, and changelog.
- Updated the quality matrix to `v0.7.2` and added the update-after-release proof to the visible coverage signals.
- Updated the README release badge to `v0.7.2`.

## Why It Matters

The repo already shows how a visitor can install one safe first skill. The next trust question is whether the library can be updated without blindly overwriting installed folders.

The new proof trail makes that maintenance path inspectable. Visitors can update from a release, test a temporary copy, reinstall deliberately, reload, and run a fake check before touching private or client-facing work.

## Safety Note

The update proof uses generic install commands and fictional first-run material. It does not include client material, private prompts, logs, credentials, account data, local machine paths, or unpublished implementation notes.
