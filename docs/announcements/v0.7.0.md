# v0.7.0

JQ AI Skills now includes a first install proof trail.

This release keeps the stable 2026 social preview unchanged. The update focuses on packaging the existing onboarding pieces into one inspectable path: clone the repo, choose one safe first skill, install it into a temporary target, verify the copied folder, run the fake candidate, and compare the result with the expected review.

## What Changed

- Added `docs/examples/first-install-proof.md`.
- Connected the proof trail to the first skill scorecard, quick reference walkthrough, install smoke-test sample, install verification guide, first-skill candidate pack, and expected review.
- Linked the first install proof from the README, `START_HERE.md`, `QUICK_REFERENCE.md`, examples index, public proof index, visitor paths, catalog, and changelog.
- Updated the README release badge to `v0.7.0`.

## Why It Matters

The repo already had the individual proof assets. The new page gives visitors a single trust path through them.

That makes the first experience easier to verify: a visitor can see not just that skills exist, but that one skill can be selected, installed, checked, run on fake material, and compared against an expected safe review.

## Safety Note

The first install proof uses a disposable target and fictional candidate files. Do not use account exports, raw logs, screenshots, unpublished notes, or real project files for the first proof run.
