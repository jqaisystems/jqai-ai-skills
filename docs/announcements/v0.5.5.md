# v0.5.5

JQ AI Skills now has an install verification guide.

This release adds `docs/guides/install-verification.md` so visitors can confirm that a skill was copied correctly, their tool reloaded it, and the first prompt produces the expected review shape before they use the skill on real work.

The stable 2026 social preview is unchanged.

## What's changed

- Added `docs/guides/install-verification.md`.
- Covered repo-root checks, skill listing, temporary install targets, copied folder checks, default target checks, reload, first prompt, cleanup, and common failure fixes.
- Linked the guide from the README, `START_HERE.md`, `INSTALL.md`, `TROUBLESHOOTING.md`, and catalog.
- Updated `CHANGELOG.md` with the v0.5.5 entry.
- Updated the README release badge to `v0.5.5`.
- Kept the stable 2026 social preview unchanged.

## Why it matters

Install docs are more useful when visitors can prove the install worked before they use a skill on real project material.

The new guide gives a small public-safe verification path: check the repo root, list skills, smoke-test a temporary target, confirm the copied folder, reload the tool, and run a first prompt with fake or public-safe files.
