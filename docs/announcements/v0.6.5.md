# v0.6.5

JQ AI Skills now has a skill library evaluation checklist.

This release adds a practical guide for deciding whether an AI skill library is clear, installable, versioned, public-safe, and ready for a test install.

The stable 2026 social preview is unchanged.

## What's changed

- Added `docs/guides/evaluate-skill-library.md`.
- Covered identity, license, install path, folder shape, safety boundaries, example quality, release discipline, first-skill choice, red flags, and a final scorecard.
- Linked the checklist from the README, start page, quick reference, public proof index, visitor paths, catalog, and changelog.
- Updated the README release badge to `v0.6.5`.
- Kept the stable 2026 social preview unchanged.

## Why it matters

The repo now gives visitors a way to evaluate the proof, not just browse it.

That makes the library easier to inspect before installation: a visitor can check whether the repo has a license, safe install path, review boundaries, sample outputs, release history, and a clear first skill to try.
