# v0.6.6

JQ AI Skills now has a first-skill scorecard for visitors who have evaluated the library and want to choose one safe first install.

This release keeps the stable 2026 social preview unchanged. The update focuses on the step after trust review: turning "this repo looks useful" into "this is the one skill I can test safely first."

## What Changed

- Added `docs/guides/first-skill-scorecard.md`.
- Added a simple `0` to `2` scorecard for job match, safe first test, output clarity, install confidence, and reuse value.
- Added tie breakers and stop conditions so visitors install one reviewed skill instead of installing everything by default.
- Linked the scorecard from the README, `START_HERE.md`, `QUICK_REFERENCE.md`, public proof index, visitor paths, examples index, evaluation checklist, skill selection guide, catalog, and changelog.
- Updated the README release badge to `v0.6.6`.

## Why It Matters

The repo already helps visitors evaluate trust, install readiness, public proof, and skill quality.

The scorecard closes the next small gap: choosing the first skill with a clear reason, a safe fake-data input, and an expected output to compare.

## Safety Note

Do not test a first skill with credentials, account exports, logs, customer data, private project notes, or local private paths. Use fake or public-safe input first, review the output, and install into the real target only after the test is clear.
