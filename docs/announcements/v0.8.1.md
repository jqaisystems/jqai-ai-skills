# v0.8.1

JQ AI Skills adds two proof-workflow skills for the part of a project that happens after the work ships: refreshing the public proof surface, checking that it renders correctly, and turning the update into trackable public evidence.

This release keeps the new skills small and focused. It does not replace `github-safe-publisher`, `case-study-writer`, or `release-announcement-writer`; it adds the operational layer that connects GitHub profile updates, social posts, proof trails, and follow-up notes.

## What's changed

### Added: public proof workflows

- `github-profile-proof-updater`: refreshes GitHub profile visuals, preserves README image paths, validates live GitHub rendering, and documents a proof trail.
- `social-campaign-proof-tracker`: plans LinkedIn/X proof campaigns, writes platform-specific copy, adds alt text, tracks published links, and prepares follow-up notes.

### Added: public-safe samples

- `docs/examples/github-profile-proof-updater-sample.md`: a fictional GitHub profile asset refresh with preserved README paths, live verification, and proof-trail update.
- `docs/examples/social-campaign-proof-tracker-sample.md`: a fictional LinkedIn/X proof campaign with copy, alt text, public links, and follow-up windows.

### Docs

- README: added prompt examples and Public Proof rows for the two new skills.
- `docs/catalog.md`: added At A Glance rows, detailed skill sections, and a Proof Campaign Handoff bundle.
- `docs/skill-quality-matrix.md`: added maturity, proof, safety, and install notes for both new skills.
- `docs/examples/README.md` and `docs/examples/public-proof-index.md`: linked the new public-safe samples.
- `QUICK_REFERENCE.md`: added common prompts and proof sample links.
- `CHANGELOG.md`: added the v0.8.1 entry.

## Safety

- Both skills are draft/review-first.
- Neither skill posts, uploads, comments, messages, commits, or pushes without explicit user approval.
- Samples use fictional links and public-safe placeholders only.
- `client-intake-triage` remains private because `client-intake-builder` and `lead-scorer` already cover adjacent public needs.
