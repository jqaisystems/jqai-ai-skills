# Social Campaign Proof Tracker Sample

This sample uses fictional public links and public-safe campaign notes.

## Input

Proof object: GitHub profile refresh

Public links:

- GitHub profile: `https://github.com/example`
- Case study: `https://example.com/systems/profile-proof-layer`

Assets:

- Workflow map image
- Three supporting proof tiles
- Short vertical video

## Expected Campaign Plan

Recommendation:

- LinkedIn gets the full visual story with a reflective post.
- X gets a short thread led by the workflow map.
- The proof trail stores published links and follow-up windows.

LinkedIn copy:

```text
I refreshed my GitHub profile into a public proof layer for AI automation work.

The goal was simple: make the profile explain what I build, how the pieces connect, and where the public proof lives.

It now connects systems, reusable skills, demos, and case studies in one place.
```

X thread:

```text
1/ I refreshed my GitHub profile into a public proof layer for AI automation work.

2/ The profile now connects systems, reusable skills, demos, and case studies.

3/ The useful part is not the visuals alone. It is the repeatable proof workflow behind them.

4/ GitHub: https://github.com/example
```

Alt text:

- Workflow map showing how profile visuals connect AI automation systems, reusable skills, and public proof.
- Tile summarizing AI automation systems built with review-first workflows.
- Tile summarizing reusable skills packaged for repeatable AI work.

Follow-up:

| Window | Action | Status |
|---|---|---|
| Same day | Reply to comments and save useful questions. | Planned |
| Next day | Add public links to the proof trail. | Planned |
| One week | Review what can become a case study or skill update. | Planned |

## Safety Notes

- Do not store private analytics exports.
- Do not invent engagement numbers.
- Do not imply client permission or measurable outcomes unless approved.
- Keep reply drafts human-reviewed.
