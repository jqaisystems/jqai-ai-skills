## Safety Boundary

These examples are educational output shapes. Before adapting them to real work:

- replace fictional details with approved public or internal-safe material;
- remove sensitive details before publishing anything;
- verify links and claims;
- keep human review gates where the workflow affects public content, outreach, releases, or marketplace listings.

Use [`../../SECURITY.md`](../../SECURITY.md) and [`../../RELEASE_CHECKLIST.md`](../../RELEASE_CHECKLIST.md) before publishing public artifacts.
